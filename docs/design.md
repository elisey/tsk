# tsk - Issue Tracker для LLM агентов

## Context

Создаём CLI инструмент для отслеживания задач, оптимизированный для использования LLM агентами. Задачи хранятся в Markdown файлах, которые пользователь может читать напрямую. Инструмент позволяет агенту видеть доступную работу, создавать задачи, обновлять статусы и управлять зависимостями.

---

## Технический стек

- **Python** + **uv** (менеджер пакетов)
- **Typer** (парсинг CLI аргументов)

---

## Архитектура хранения

### Директория `.tsk/`
- Поиск `.tsk/` рекурсивно вверх по дереву директорий (git-стиль)
- Требуется явный `tsk init` перед использованием

### Файловая структура (после init)
```
.tsk/
├── todo.md
├── in_progress.md
└── closed.md
```

Один файл = один статус. При смене статуса задача перемещается между файлами.

### ID задач
- Числовой auto-increment (1, 2, 3...)
- Вычисляется динамически: `max(id)` среди всех задач + 1
- Не требует отдельного файла счётчика

---

## Формат Markdown

Используем ## заголовки с секциями. Порядок полей гибкий - парсер находит нужные.

```markdown
## Issue 5

**Title:** Fix authentication bug
**Priority:** 1
**Depends on:** 3, 7
**Created:** 2024-03-15T14:30:00
**Updated:** 2024-03-15T16:45:00

### Description

Многострочное описание задачи.
Может содержать любой текст.
```

Для closed.md добавляется поле `**Closed:** 2024-03-15T18:00:00`

### Форматы данных
- **Даты:** ISO 8601 (`2024-03-15T14:30:00`)
- **Приоритеты:** числа 0, 1, 2 (0 = критический, 1 = средний, 2 = низкий)
- **Зависимости:** только `depends_on` (односторонние ссылки)

## CLI команды

### Инициализация
```bash
tsk init              # Создаёт .tsk/ с пустыми todo.md, in_progress.md, closed.md
```

### Просмотр задач
```bash
tsk ready             # Задачи без активных блокеров, сортировка: priority → ID
tsk list              # Все задачи
tsk list --status=todo|in_progress|closed|open  # Фильтр (open = todo + in_progress)
tsk show <id>         # Полная информация о задаче
```

### Управление задачами
```bash
tsk create --title="..." [--description="..."] [--priority=1]
tsk update <id> --status=... --priority=... --title="..." --description="..."
tsk close <id> [<id2> ...]   # Закрыть одну или несколько задач
```

### Зависимости
```bash
tsk dep add <issue> <depends-on>    # Задача зависит от другой
tsk dep remove <issue> <depends-on> # Убрать зависимость
```

### Справка
```bash
tsk --help            # Стандартная справка Typer
tsk help              # Расширенная справка с workflow и примерами (для инъекции в LLM контекст)
```

## Поведение

### Обязательные поля
- **create:** только `--title`
- **Дефолт priority:** 1 (средний)

### Валидация статусов
- Любые переходы разрешены (todo → closed, closed → todo)
- Переоткрытие закрытой задачи разрешено через `tsk update <id> --status=todo`

### Валидация зависимостей
- **Циклы:** запрещены при создании (проверка при `tsk dep add`)
- **Self-reference:** `tsk dep add 5 5` - ошибка
- **Несуществующий блокер:** `tsk dep add 1 99` - ошибка (строгая проверка)
- **Дубликат:** повторный `tsk dep add 5 3` - ошибка
- **Закрытие заблокированной:** разрешено молча (зависимости не проверяются при close)

### Runtime вычисление блокировок
- При `tsk ready` вычисляем динамически: задача заблокирована если хотя бы один из её `depends_on` не в статусе `closed`
- Связи хранятся даже после закрытия блокера (для истории)

### Конкурентность
- Игнорируем. Предполагаем один агент в момент времени.

## Вывод

### Формат
- **Human-readable** текст (не JSON)
- **Компактный список** для `tsk list/ready`:
  ```
  #5 [P0] Fix auth bug
  #7 [P1] Add tests
  ```

### Сортировка
- Фиксированная: priority → ID
- P0 выше всех, при равном приоритете - меньший ID первый

### Пустое состояние
```
No tasks found
```
(без подсказок о командах)

### Успешные операции
- Молчание (Unix стиль)

### Ошибки
```
Error: Issue 5 not found
Hint: Run 'tsk list' to see all issues
```
- Текст + подсказка как исправить
- Exit code: 0 = успех, 1 = любая ошибка

### tsk show <id>
Полная информация:
- ID, Title, Priority, Status
- Description (полный текст)
- Dependencies (ID + title + status для каждой)
- Timestamps (created, updated, closed)

### Множественные операции
- `tsk close 1 2 99`: частичное выполнение
  - Закрыть 1 и 2
  - Вывести ошибку для 99

## Парсинг и ручное редактирование

### Поддержка ручного редактирования
- **Полностью поддерживаем** редактирование .tsk/*.md файлов
- Формат должен быть толерантным к вариациям

### Битые файлы
- Если парсинг сломался - **краш с ошибкой** (exit 1)
- Не пытаемся восстановить

### Пропущенные поля в файле
- Priority: дефолт P1

---

## Закрытые задачи

- **Хранить всё** в closed.md навсегда
- Нет авто-очистки или архивации
- Нет команды delete

---

## Git интеграция

- Нет автоматической интеграции
- Пользователь сам добавляет .tsk/ в git или .gitignore
